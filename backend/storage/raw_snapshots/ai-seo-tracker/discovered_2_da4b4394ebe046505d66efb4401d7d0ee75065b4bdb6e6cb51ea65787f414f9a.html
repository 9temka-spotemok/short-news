Why Show Content Only to AI Bots?

AI search engines like ChatGPT, Perplexity, and Claude need structured information to understand and cite your content. But adding extensive AI-optimized content can clutter your site for human visitors.

The <LLMOnly /> component solves this by:

  • Improving AI visibility without affecting user experience
  • Providing structured data that AI models can easily parse
  • Keeping pages clean for human visitors
  • Optimizing for AI SEO without traditional SEO penalties

The Complete LLMOnly Component

Here's the full implementation using Next.js 15 server components:

components/ui/llm-only.tsx
import { headers } from 'next/headers'
 
const LLM_USER_AGENTS = [
  'gptbot',
  'chatgpt-user',
  'oai-searchbot',
  'claudebot',
  'anthropic-ai',
  'claude-web',
  'mistralai-user',
  'bytespider',
  'cohere-ai',
  'perplexitybot',
  'google-extended',
  'bard',
  'gemini',
  'deepseekbot',
  'deepseek-r1',
  'grokbot',
  'xai',
  'bingbot',
  'amazonbot',
  'duckassistbot',
  'ai2bot',
  'ccbot',
  'omgili',
  'diffbot',
  'facebookbot',
  'meta-externalagent',
  'youbot',
  'applebot-extended',
]
 
export async function LLMOnly({ children }: { children: React.ReactNode }) {
  const headersList = await headers()
  const userAgent = headersList.get('user-agent')?.toLowerCase() || ''
 
  // Check for AI bot user agents
  const isAIBot = LLM_USER_AGENTS.some(agent => userAgent.includes(agent))
 
  if (!isAIBot) {
    return null
  }
 
  return <div className="llm-only-content">{children}</div>
}

How It Works

1. User Agent Detection

The component checks the incoming request's user agent against a comprehensive list of known AI bot identifiers:

  • OpenAI: gptbot, chatgpt-user, oai-searchbot
  • Anthropic: claudebot, anthropic-ai, claude-web
  • Google: google-extended, bard, gemini
  • Perplexity: perplexitybot
  • Other AI platforms: DeepSeek, Grok, Cohere, and more

2. Server-Side Rendering

Using Next.js 15's async server components, the detection happens at render time:

components/ui/llm-only.tsx
const headersList = await headers()
const userAgent = headersList.get('user-agent')?.toLowerCase() || ''

3. Conditional Rendering

If an AI bot is detected, the content renders. Otherwise, it returns null:

Why Dynamic Pricing Sliders Are Problematic

Many SaaS companies use dynamic pricing sliders that let users adjust features and see real-time pricing. However, these sliders are often:

  • JavaScript-dependent: Only work after client-side code loads
  • State-based: Show different prices based on user interaction
  • Not in initial HTML: Missing from the static content AI crawlers see

Solution: Use the LLMOnly component to provide static pricing information that AI can read, while keeping your interactive slider for human users.

The result? AI gets incomplete information and cites competitors instead:

This is exactly why the LLMOnly component is crucial - without it, AI systems will find pricing information from secondary sources (often competitors) rather than your official pricing page.

Implementation for Webflow & Framer

Don't use Next.js? You can still optimize for AI crawlers using these no-code platforms.

Webflow Implementation

Webflow doesn't have server-side rendering, but you can use CSS + Custom Code to hide content from humans while keeping it accessible to AI crawlers.

Method 1: Pure CSS (Simplest)

  1. Create a div block with a class name llm-only
  2. Add your AI-targeted content inside it (headings, text blocks, lists)
  3. Add this Custom Code to Page Settings → Custom Code → Before </body> tag:
<style>
  .llm-only {
    /* Hide from visual display but keep in DOM for AI crawlers */
    position: absolute !important;
    left: -9999px !important;
    width: 1px !important;
    height: 1px !important;
    overflow: hidden !important;
    clip: rect(1px, 1px, 1px, 1px) !important;
  }
</style>

  ├─ Paragraph: "Detailed product information for AI..."

  └─ Rich Text:
      • Starter: $29/month - 5 users, 10GB storage, basic support
      • Pro: $99/month - 25 users, 100GB storage, priority support
      • Enterprise: $299/month - Unlimited users, 1TB storage, 24/7 support

For true AI bot detection in Webflow, add this to Page Settings → Custom Code → Before </body> tag:

  • ⚠️ Content visible in page source

Framer Implementation

Framer offers multiple approaches depending on your comfort level with code.

Method 1: Code Override (Recommended)

  1. Open the Code tab in your Framer project
  2. Create a new file: LLMOnly.tsx
  3. Paste this code:
import { Override } from 'framer'
 
export function LLMOnly(): Override {
  return {
    style: {
      position: 'absolute',
      left: '-9999px',
      width: '1px',
      height: '1px',
      overflow: 'hidden',
      clip: 'rect(1px, 1px, 1px, 1px)',
    },
  }
}
  1. Apply the override to a Frame containing your AI-only content
  2. In the properties panel: Override → Select LLMOnly

Method 2: Code Component

  1. Create a new Code Component: LLMOnlyContent.tsx
export default function LLMOnlyContent({ children }) {
  return (
    <div
      style={{
        position: 'absolute',
        left: '-9999px',
        width: '1px',
        height: '1px',
        overflow: 'hidden',
        clip: 'rect(1px, 1px, 1px, 1px)',
      }}
      className="llm-only-content"
    >
      {children}
    </div>
  )
}
  1. Drag the component onto your canvas
  2. Add text components as children with your AI-optimized content

Method 3: Custom Code (Site-Wide)

Add this to Site Settings → Custom Code → End of <body>:

<script>
  document.addEventListener('DOMContentLoaded', function () {
    const style = document.createElement('style')
    style.textContent = `
      .llm-only {
        position: absolute !important;
        left: -9999px !important;
        width: 1px !important;
        height: 1px !important;
        overflow: hidden !important;
        clip: rect(1px, 1px, 1px, 1px) !important;
      }
    `
    document.head.appendChild(style)
  })
</script>

Then add llm-only as a custom class to any Frame you want hidden from humans.

Why Use position: absolute Instead of display: none?

The key difference:

/* ❌ Some AI crawlers might skip this */
.llm-only {
  display: none;
}
 
/* ✅ Content stays in DOM and is accessible to crawlers */
.llm-only {
  position: absolute;
  left: -9999px;
  clip: rect(1px, 1px, 1px, 1px);
}

Using position: absolute with off-screen positioning ensures:

  • Content remains in the HTML structure
  • AI crawlers can parse and index it
  • Screen readers can still access it (accessibility benefit)
  • Doesn't trigger "hidden content" penalties

Advanced: Edge Function Approach (All Platforms)

For true server-side detection on Webflow or Framer, use Cloudflare Workers as a reverse proxy:

  1. Set up Cloudflare Workers on your domain
  2. Add bot detection logic:
export default {
  async fetch(request) {
    const userAgent = request.headers.get('user-agent')?.toLowerCase() || ''
 
    const isAIBot = ['gptbot', 'claudebot', 'perplexitybot', 'google-extended'].some(agent => userAgent.includes(agent))
 
    // Fetch from your Webflow/Framer site
    const response = await fetch(request)
    let html = await response.text()
 
    if (isAIBot) {
      // Inject or unhide AI-only content
      html = html.replace(/class="llm-only" style="display: none;"/, 'class="llm-only"')
    }
 
    return new Response(html, response)
  },
}

This gives you Next.js-level bot detection on any platform, but requires additional infrastructure.
